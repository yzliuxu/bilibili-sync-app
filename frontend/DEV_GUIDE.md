## 🏗️ 代码结构（开发模式）

### 开发代码条件编译
```javascript
// 这些代码只在开发模式（npm run dev）执行
if (import.meta.env.DEV) {
  // 生成模拟数据
  const mockTasks = generateMockTasks();
  
  // 跳过 Token 验证
  console.log("🔓 [DEV MODE] 开发模式已启用");
  
  // 显示开发模式标识
  return (
    <div>🔓 开发模式 - 正在使用模拟数据</div>
  );
}

// 生产环境代码
// 正常进行 API 调用和 Token 验证
```

### Tree-shaking 自动清理
- ✅ `generateMockTasks()` 函数 - 生产环境移除
- ✅ 所有 `import.meta.env.DEV` 分支 - 编译时删除
- ✅ 开发模式 UI 提示 - 编译时删除

## 📦 编译时的行为

1. **所有开发代码被移除**

## 📝 常见问题

### Q: 能否在生产环境使用开发数据？
**A:** 不能。生产编译时会自动移除所有开发代码。这通过 Vite 的 Tree-shaking 实现。

### Q: 如何确认生产环境已正确移除开发代码？
**A:** 使用上面提供的 `grep` 命令检查编译后的 JS 文件。

### Q: 开发数据何时加载？
**A:** 在 `npm run dev` 启动时，hooks 初始化时加载。

### Q: 为什么要跳过 Token 验证？
**A:** 方便本地开发和测试，无需运行后端服务。
